# LinguaAI Website Brief — Clean Source of Truth

This folder replaces the older, over-specified briefing files.

## Why this exists

The previous project files mixed product briefing, copywriting, layout decisions, design systems, technical implementation details, and strict UI prescriptions in a way that created unnecessary constraints.

This clean set keeps only what should currently be fixed:

- what the project is
- what the website is about
- who it is for
- what the website should achieve
- which tech stack should be used
- which decisions must remain open

## Source-of-truth order

1. `01_project_briefing.md`
2. `02_tech_stack.md`
3. `03_agent_guidelines.md`
4. `04_open_questions.md`

## Important

Older files such as `BRIEF.md` and `WEBSITE-BLUEPRINT.md` may still contain useful product context, but they should **not** be treated as binding for:

- visual style
- color system
- typography
- section order
- component design
- page architecture
- exact copy
- interaction patterns
- implementation details outside the approved stack

If older files conflict with this folder, this folder wins.
