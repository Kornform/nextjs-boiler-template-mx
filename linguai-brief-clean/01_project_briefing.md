# Project Briefing

## Project Type

Marketing website / web design project for a WordPress plugin.

## Working Product Name

`LinguAI for WPML`

This is a working name, not a locked final brand name.
The naming pattern `NAME for WPML` is currently relevant, but the final product name may still change.

## Product Summary

`LinguAI for WPML` is a WordPress plugin for websites that already use WPML.
It helps users translate content with AI while keeping control over terminology, tone, context, and workflow.

The plugin is positioned as a more flexible and controllable alternative to rigid or expensive translation workflows.

## Website Goal

The website should explain the plugin clearly and make the product feel easy to understand, useful, and trustworthy.

Primary business goal:
- drive free plugin adoption / downloads

Secondary business goal:
- create interest in a Pro version or paid plan

## Core Audience

Primary audience:
- WordPress users already using WPML
- agencies managing multilingual client sites
- site owners who want less manual translation work
- users who care about quality, control, and SEO-sensitive output

## What the Website Must Communicate

The website should make these points understandable:

- this is a plugin for WPML users
- it helps automate translation work
- users keep control over the output
- the workflow should feel simpler than common alternatives
- advanced capabilities exist, but usability should remain clear

## Product Themes

These are valid product themes for messaging and structure:

- easier multilingual workflow
- more control over translations
- reusable translation setup
- glossary / terminology control
- prompt / context control
- better fit for SEO-sensitive content
- less repetitive manual work

These are themes, not fixed homepage copy.

## Scope for This Phase

In scope:
- marketing website
- positioning
- structure
- design exploration
- content strategy
- implementation within the approved stack

Out of scope for now:
- backend
- dashboard product UX
- account system
- checkout implementation details
- server architecture
- plugin internals

## Deliberately Left Open

The following are intentionally **not fixed** in this briefing:

- final visual style
- final branding direction
- final colors
- final typography
- final page architecture
- exact homepage section order
- exact copy
- exact CTA wording
- exact pricing presentation
- exact interaction patterns
- final content depth per page

## Working Direction

The website should be able to support both of these without locking either one too early:

- a simpler, approachable marketing direction
- a more premium but still clear product presentation

Any future design work should derive from product goals first, not from a pre-written visual system.
